ISO/IEC 11072

ISO/IEC 11072 – Graphical Kernel System (GKS) functional description for 2D computer graphics (1985/legacy edition)

Overview

ISO/IEC 11072 (commonly known as GKS – Graphical Kernel System) is the international standard defining a 2D graphics application programming interface and functional specification. First published in 1985 (with amendments and technical corrigenda through the 1990s), it provides a device-independent abstraction layer for 2D graphics output, input, and control. In March 2026, ISO/IEC 11072 remains formally current but is considered a **legacy/historical standard**. It has been largely superseded by modern APIs (OpenGL, Vulkan, DirectX, WebGL, Canvas2D, SVG) and file formats (SVG, PDF). GKS is still referenced in academic graphics education, legacy system maintenance, formal specifications in safety-critical or regulated domains (e.g., defense, aerospace, medical visualization), and historical context for graphics standards evolution.

Key Value: ISO/IEC 11072 (GKS) provides a foundational, device-independent 2D graphics model and API specification — historically enabling portable graphics applications, standardized 2D output/input, and vendor-neutral interoperability in early computer graphics systems, and still relevant for legacy maintenance and academic/formal graphics reference.

Core Features & Model (1985 Edition – 2026)

Key Abstractions

  • Workstations (abstract display devices)
  • Output primitives (polylines, polygons, text, markers, filled areas)
  • Attributes (line type/width/color, fill style, text font/height)
  • Segments (retained display lists)
  • Normalization transformation & viewing pipeline

Input Model

  • Logical input devices (locator, stroke, valuator, choice, pick, string)
  • Input modes (request, sample, event)
  • Event queue handling

Security & Control

  • Workstation control (open/close, activate/deactivate)
  • Error handling & inquiry functions
  • Metafile support for picture storage

Key Adoption & Trends in 2026

Legacy & Regulated Use

  • Maintenance of legacy GKS-based systems (defense, aerospace, CAD)
  • Referenced in formal education & historical graphics papers
  • Occasional use in safety-critical visualization requiring certified 2D output

Superseded by Modern APIs

  • OpenGL, Vulkan, DirectX, WebGL, Canvas2D, SVG dominant
  • MP4/H.264 (ISO 14496) & DASH (ISO 23009) for video/graphics
  • No active development or new profiles since 1990s

Academic & Reference Role

  • Used in computer graphics textbooks & courses
  • Historical reference for device-independent graphics concepts
  • Occasional mapping to modern vector graphics (SVG)

Best Practices for ISO/IEC 11072 Reference (2026)

  • Document GKS usage in legacy systems
  • Ensure device emulation for modern hardware
  • Plan gradual migration to modern APIs (OpenGL/WebGL)

Education & Formal Use

  • Use for teaching device-independent graphics principles

  • Reference in formal specifications & certification
