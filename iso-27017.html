ISO/IEC 27017

ISO/IEC 27017 – code of practice for information security controls in cloud services (2015 edition)

Overview

ISO/IEC 27017:2015 is the international standard providing specific guidance and additional controls for securing information in cloud computing environments. It extends ISO/IEC 27002 with 37 cloud-specific controls and implementation advice for both cloud service providers and cloud service customers. In March 2026, ISO/IEC 27017 remains the globally recognized reference for cloud security controls, frequently required in cloud contracts, data processing agreements (DPAs), third-party risk assessments, and compliance frameworks (GDPR Art. 28, NIS2, FedRAMP, CSA CCM, ISO 27001 certification mappings). It covers shared responsibility models, virtualization security, cloud-specific access controls, data segregation, encryption, incident management, and supplier relationships in cloud contexts. Major cloud providers (AWS, Microsoft Azure, Google Cloud, Oracle, Alibaba) publish ISO 27017 compliance attestations and mappings.

Key Value: ISO/IEC 27017 offers clear, auditable cloud-specific security guidance that clarifies shared responsibilities, reduces cloud-related risks, supports secure cloud adoption, and helps demonstrate due diligence to regulators, auditors, customers, and partners in public, private, and hybrid cloud environments.

  • CLD.6.2.1 – Supplier relationships in cloud supply chain

Integration with ISO 27001/27002

  • Extends ISO 27002 controls with cloud-specific guidance
  • Supports joint certification (27001 + 27017)
  • Used in Statement of Applicability (SoA) for cloud services
