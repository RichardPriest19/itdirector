FTP

File Transfer Protocol – legacy protocol for transferring files over TCP networks

Overview

FTP (File Transfer Protocol) is a standard network protocol (defined in RFC 959, 1985) used for transferring files between a client and server over a TCP network. It operates in two modes (active/passive) and uses separate control (port 21) and data connections. In March 2026, **plain FTP is considered obsolete and insecure** — credentials and data are transmitted in clear text, making it vulnerable to interception, credential theft, and man-in-the-middle attacks. It has been almost entirely replaced by secure alternatives: **SFTP** (SSH File Transfer Protocol), **FTPS** (FTP over TLS), **SCP**, **WebDAV over HTTPS**, **S3-compatible APIs**, **rsync over SSH**, and modern cloud file sync/share services. FTP is now only encountered in very old industrial equipment, legacy mainframe systems, or misconfigured legacy workflows — and its continued use is strongly discouraged.

Key Value (historical): FTP was once the simplest and most universal way to move files across networks — but in 2026 its only remaining “value” is as a reminder of insecure-by-design protocols. All modern organizations should have fully migrated to encrypted, secure file transfer methods.

Core Characteristics & Legacy Status (2026)

FTP Fundamentals

  • Control channel: TCP port 21 (commands)
  • Data channel: dynamic ports (active mode) or port 20 (passive mode)
  • Clear-text authentication & data transfer
  • Active vs Passive mode (firewall traversal challenges)
  • ASCII vs Binary transfer modes

Security Issues

  • Credentials transmitted in plain text
  • No encryption of data in transit
  • Vulnerable to eavesdropping, replay, and MITM attacks
  • No integrity protection or modern authentication

Modern Secure Alternatives

  • SFTP (SSH File Transfer Protocol) – port 22, encrypted
  • FTPS (Explicit/Implicit) – FTP over TLS, port 990/21
  • SCP / rsync over SSH – simple secure file copy
  • WebDAV over HTTPS, S3 APIs, Azure Blob, Google Cloud Storage
  • Secure file share platforms (Box, Dropbox Business, OneDrive for Business)

Key Trends & Status in 2026

Legacy & Industrial Use

  • Still found in old manufacturing equipment, mainframes, embedded systems
  • Common in air-gapped or isolated legacy environments
  • Gradual decommissioning & replacement programs ongoing

Security & Compliance Pressure

  • Plain FTP explicitly prohibited by PCI-DSS, HIPAA, GDPR, NIS2
  • Cyber insurance policies often exclude FTP-related incidents
  • Regulatory audits flag FTP as high-risk

Migration & Replacement

  • Most organizations completed FTP → SFTP/FTPS migration by 2024–2025
  • Cloud-native file transfer (S3, Azure Blob, Google Cloud Storage APIs) dominant
  • Zero-trust file access & encrypted channels mandatory

Best Practices for FTP-Related Situations (2026)

Migration Strategy

  • Identify all FTP usage (scans, logs, firewall rules)
  • Prioritize migration to SFTP, FTPS or cloud-native transfer
  • Replace legacy FTP servers with secure alternatives (AWS Transfer Family, Azure Files, etc.)

Interim Mitigation (if FTP cannot be removed immediately)

  • Force FTPS (Explicit TLS) & disable plain FTP
  • Restrict to internal/VPN-only access
  • Implement strong credential rotation & monitoring

Security & Monitoring

  • Block outbound FTP (port 21) at firewall unless explicitly justified
  • Monitor for FTP traffic & alert on unauthorized use
  • Conduct regular legacy protocol audits

IT Director Responsibilities

  • Champion complete elimination of plain FTP across the organization
  • Approve FTP migration roadmap & secure alternatives
  • Ensure zero-trust & encrypted file transfer policies
  • Oversee legacy protocol scans & risk assessments
  • Monitor for unauthorized FTP usage & compliance violations
  • Drive SFTP/FTPS/cloud-native adoption & training
  • Align file transfer strategy with regulatory & cyber insurance requirements
