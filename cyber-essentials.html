Cyber Essentials

UK government-backed cybersecurity certification scheme for fundamental security controls

Overview

Cyber Essentials is a UK government-backed cybersecurity scheme created by the Department for Science, Innovation and Technology (DSIT) in partnership with the National Cyber Security Centre (NCSC). It provides organizations with a straightforward, evidence-based approach to cyber security by focusing on implementing five fundamental controls that protect against the most common cyber attacks. The scheme has become widely recognized as best practice and is endorsed by government, industry, and certification bodies worldwide.

Key Benefit: Cyber Essentials helps organizations reduce cyber risk from common threats and demonstrates security commitment to customers, partners, and regulators. Increasingly required for government contracts and public sector suppliers.

The Five Core Controls

Cyber Essentials focuses on implementing five fundamental controls that protect against the vast majority of common cyber threats. These controls target the most frequently exploited vulnerabilities:

1. Boundary Firewalls and Internet Gateways

Protect the network perimeter from unauthorized access and malicious traffic.

  • Requirement: Deploy firewalls at the network boundary (between organization and internet)
  • Configuration: Default deny incoming traffic; explicitly allow necessary connections
  • Rules: Block unnecessary protocols, services, and ports
  • Monitoring: Log and monitor firewall activity for suspicious patterns
  • Management: Regularly review and update firewall rules
  • Protection Against: Unauthorized network access, DDoS attacks, malicious traffic

2. Secure Configuration

Remove unnecessary services, apply security settings, and harden systems against attack.

  • Remove Unnecessary Services: Disable services not required for business function
  • Remove Unnecessary Software: Uninstall applications not actively used
  • Apply Security Settings: Configure operating system security options
  • Secure Defaults: Change default passwords and settings from vendor defaults
  • Regular Review: Periodically review configurations to ensure compliance
  • Documentation: Document all configurations and deviations
  • Protection Against: Exploitation of unnecessary services, configuration weaknesses

3. User Access Control

Restrict access to systems and data based on user role and business need.

  • Administrative Privileges: Restrict who has admin/privileged access
  • Standard User Accounts: Users should not have admin rights for daily work
  • Principle of Least Privilege: Grant only necessary permissions for each role
  • Access Reviews: Regularly review and revoke unnecessary access
  • Account Management: Disable or delete unused accounts
  • Separation of Duties: Prevent individuals from having conflicting roles
  • Protection Against: Unauthorized access, privilege escalation, insider threats

4. Malware Protection

Deploy anti-malware tools and keep them current to detect and prevent malicious software.

  • Deployment: Install anti-virus/anti-malware on all computers and servers
  • Coverage: Protect all endpoints, not just user workstations
  • Real-Time Scanning: Enable continuous monitoring of system activity
  • Regular Updates: Keep malware definitions current (at least weekly)
  • Periodic Scanning: Conduct regular full system scans
  • Quarantine and Remediation: Isolate and remove infected files
  • User Restrictions: Prevent users from disabling protection
  • Protection Against: Viruses, trojans, ransomware, spyware

5. Patch Management

Keep operating systems and applications up-to-date with security patches.

  • Operating Systems: Apply security patches to Windows, Linux, macOS
  • Applications: Patch web browsers, email clients, office software, etc.
  • Firmware: Update device firmware (routers, switches, printers)
  • Timely Application: Apply critical patches within 2 weeks, others within 1 month
  • Testing: Test patches in non-production before widespread deployment
  • Automation: Use automatic update mechanisms where possible
  • Monitoring: Track what systems have been patched
  • Protection Against: Exploitation of known vulnerabilities

Cyber Essentials vs Cyber Essentials Plus

Cyber Essentials (Self-Assessment)

  • Assessment Type: Self-assessment questionnaire (SAQ)
  • Verification: Organization completes questionnaire; no independent verification
  • Cost: Lower cost (approximately £100-300)
  • Timeline: Quick certification (1-2 months typical)
  • Credibility: Demonstrates self-assessed compliance
  • Best For: Organizations starting cybersecurity journey, internal focus

Cyber Essentials Plus (Verified Assessment)

  • Assessment Type: Independent technical assessment by NCSC-approved assessor
  • Verification: Third-party assessor tests and verifies controls
  • Cost: Higher cost (approximately £1,000-5,000+)
  • Timeline: Longer process (2-3 months typical)
  • Credibility: High credibility with external stakeholders
  • Best For: Government suppliers, customer assurance, competitive advantage

Which Should You Choose?

Choose Cyber Essentials if: You're starting cybersecurity implementation, need cost-effective approach, have lower external pressure, focus is internal improvement

Choose Cyber Essentials Plus if: You bid on government contracts, need to assure customers of controls, require external credibility, want competitive advantage in tenders

Cyber Essentials Certification Process

Stage 1: Organization Registration

  • Register with NCSC-approved scheme operator
  • Provide basic organization information
  • Identify scope of certification (systems and locations included)
  • Pay certification fee

Stage 2: Self-Assessment Questionnaire (SAQ)

  • Complete detailed questionnaire about security controls
  • Questions cover the five core control areas
  • Answer based on current implementation
  • Provide evidence or justification for answers
  • Timeline: 1-2 weeks to complete

Stage 3: Submission and Review

  • Submit completed SAQ to scheme operator
  • Scheme operator reviews responses
  • May request clarification or additional evidence
  • Review timeline: 2-4 weeks

Stage 4: Certification Decision

  • If compliant: Certification issued
  • Certificate valid for 12 months
  • Can display Cyber Essentials logo
  • If non-compliant: Receive feedback and opportunity to remediate

Stage 5: Annual Renewal

  • Annual recertification required
  • Complete updated SAQ
  • Demonstrate continued compliance
  • Certificate renewed for another 12 months

Cyber Essentials Scope and Applicability

Who Should Implement Cyber Essentials?

  • Organizations handling sensitive or confidential data
  • Government and critical infrastructure operators
  • Organizations bidding on government contracts
  • Organizations handling customer financial data
  • Organizations in regulated industries
  • Any organization wanting to improve cybersecurity posture

Scope Determination

Define what systems and locations are included in certification:

  • All locations or specific locations
  • All systems or specific critical systems
  • Cloud systems or on-premises only
  • Remote working environments
  • Third-party managed systems

Implementation Considerations

Small Organizations (20-50 staff)

  • Effort: 2-4 weeks to implement controls
  • Cost: Minimal (mostly internal effort)
  • Challenges: Limited IT staff, resource constraints
  • Timeline to Certification: 2-3 months

Medium Organizations (50-500 staff)

  • Effort: 4-8 weeks to implement controls
  • Cost: Moderate (tools, possible external help)
  • Challenges: Complex environments, multiple departments
  • Timeline to Certification: 3-4 months

Large Organizations (500+ staff)

  • Effort: 8-12 weeks to implement controls
  • Cost: Higher (tools, external expertise)
  • Challenges: Complex IT environment, legacy systems
  • Timeline to Certification: 4-6 months

Benefits of Cyber Essentials Certification

  • Reduced risk from common cyber attacks (malware, unpatched systems, weak passwords)
  • Demonstrates security commitment to customers and partners
  • Improved customer confidence in organization's security
  • Competitive advantage in government tenders and contracts
  • Structured approach to cybersecurity implementation
  • Improved employee awareness of security responsibilities
  • Foundation for more advanced security programs
  • Alignment with government cybersecurity expectations
  • Cost-effective security improvement
  • Recognition by government and industry bodies

Common Cyber Essentials Implementation Issues

  • Incomplete Firewall Configuration: Not blocking all unnecessary ports and protocols
  • Legacy Systems: Older systems not supporting modern security settings
  • Malware Protection Gaps: Some systems or users excluded from protection
  • Delayed Patching: Patches not applied within required timeframes
  • Excessive Privileges: Users having admin rights for daily tasks
  • Weak Access Controls: Not properly controlling who has access to what
  • Documentation Gaps: Not maintaining evidence of control implementation
  • Scope Confusion: Unclear about what systems should be covered

IT Director Responsibilities

  • Lead Cyber Essentials implementation and assessment process
  • Ensure firewall configuration meets requirements
  • Implement system hardening and secure configuration standards
  • Deploy and maintain malware protection across all systems
  • Establish and manage patch management process
  • Implement access control and privilege management
  • Maintain documentation of control implementation
  • Provide evidence for SAQ questionnaire
  • Ensure annual recertification
  • Communicate importance of security to staff
  • Address any assessment findings or gaps

Cyber Essentials for Government Suppliers

Government Contract Requirements

Many UK government contracts require suppliers to have Cyber Essentials certification (or equivalent). This has become standard for:

  • Direct government service providers
  • Cloud service providers to government
  • Organizations handling government data
  • Critical infrastructure organizations

Government Procurement Advantage

Having Cyber Essentials certification:

  • Makes organization eligible for government tenders
  • Demonstrates compliance with government security expectations
  • Provides competitive advantage in procurement evaluation
  • May be mandatory for certain contract types

Cyber Essentials and Other Standards

Cyber Essentials as Foundation

Cyber Essentials provides a foundation that supports implementation of more comprehensive security frameworks:

  • Cyber Essentials → Cyber Essentials Plus: Enhanced verification of controls
  • Cyber Essentials → ISO 27001: More comprehensive ISMS requirements
  • Cyber Essentials → NIST CSF: Broader risk management framework
  • Cyber Essentials + GDPR: Combined approach to data protection
  • Cyber Essentials + PCI-DSS: Foundation for payment card security

Integration Approach

Organizations often implement Cyber Essentials first, then expand to more comprehensive standards. This phased approach allows:

  • Manageable implementation effort
  • Quick wins and early value demonstration
  • Foundation for more advanced programs
  • Improved organizational security maturity over time

Getting Started with Cyber Essentials

  • Understand business drivers (government contracts, customer requirements, risk management)
  • Assess current security posture against five core controls
  • Identify gaps in firewall, configuration, access control, malware protection, patching
  • Develop implementation plan with timelines
  • Allocate resources and budget
  • Implement controls across all systems in scope
  • Document control implementation and evidence
  • Register with NCSC-approved scheme operator
  • Complete Self-Assessment Questionnaire
  • Submit for assessment and certification
  • Plan for annual recertification
