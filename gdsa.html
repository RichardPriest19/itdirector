GDSA

GIAC Defensible Security Architecture – SANS/GIAC certification for enterprise network security architecture and design

Overview

The GIAC Defensible Security Architecture (GDSA) is awarded by GIAC and is based on the SANS SEC530 Defensible Security Architecture and Engineering course. It validates skills in designing and building enterprise security architectures that are resilient, observable, and defensible against modern threats — applying Zero Trust principles, network segmentation, and layered defensive controls.

Key Value: GDSA is the definitive certification for security architects who design network and infrastructure security — covering the architectural principles, design patterns, and technical controls required to build environments that are genuinely difficult to compromise and easy to monitor.

Core Competency Areas

Defensible Architecture Principles

  • Apply Zero Trust Architecture (ZTA) principles — never trust, always verify, assume breach
  • Design network segmentation using VLANs, microsegmentation, and software-defined networking
  • Implement defence in depth with overlapping preventive, detective, and corrective controls
  • Apply the principle of least privilege across network paths, identities, and data flows

Network Security Design

  • Design secure perimeter architecture — next-generation firewalls, DMZs, and secure proxy services
  • Implement network detection and response (NDR) using traffic analysis and anomaly detection
  • Apply DNS security — DNSSEC, DNS filtering, and DNS-based threat detection
  • Design secure remote access — ZTNA, SSL VPN, and privileged access workstations (PAWs)

Identity & Access Architecture

  • Design IAM architectures using Active Directory, Azure AD/Entra, and federated identity
  • Apply Privileged Access Management (PAM) — just-in-time access, jump servers, and session recording
  • Implement MFA, phishing-resistant authentication (FIDO2/passkeys), and conditional access policies
  • Design network access control (NAC) for device trust and posture assessment

Detection & Response Architecture

  • Design SIEM and security monitoring architectures for comprehensive log ingestion and correlation
  • Implement endpoint detection and response (EDR/XDR) across the enterprise estate
  • Apply deception technologies — honeypots, honeytokens, and canary files
  • Design security automation and orchestration (SOAR) for scalable incident response

Exam Details

  • 75 questions over 2 hours — open-book (notes permitted)
  • Minimum passing score: 63%
  • Based on SANS SEC530 Defensible Security Architecture and Engineering
  • Certification valid for 4 years; renewed via GIAC CPE credits

IT Director Responsibilities

  • Require GDSA for security architects and senior network security engineers designing enterprise security infrastructure
  • Use GDSA standards as the architectural benchmark for new network and security infrastructure projects
  • Engage GDSA-qualified architects to review and validate existing network segmentation and detection capabilities
  • Position GDSA as the architecture complement to operational certifications such as GCIH and GCFA in the security team
