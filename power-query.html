Power Query

Power Query – powerful, low-code data connectivity & transformation engine across Microsoft Fabric, Power BI, Excel

Overview

Power Query (M language) is Microsoft's unified data preparation engine for connecting to, cleaning, shaping, and transforming data from hundreds of sources. In March 2026, it is central to **Microsoft Fabric** (Dataflows Gen2, Notebooks), **Power BI** (Get Data, transformations), **Excel** (Get & Transform), and programmatic execution via REST API (preview). Key evolutions include AI-assisted steps (Copilot in Fabric/Power BI), relative references in Dataflows, recent data quick access, new connectors (Vertica ODBC), and enhanced refresh/performance in Fabric-centric workflows. Power Query enables repeatable, refresh-friendly logic with no-code UI and advanced M code extensibility.

Key Value: Power Query delivers consistent, governed data prep across tools, reduces manual ETL effort, supports AI-assisted transformations, and powers fast, reliable analytics — making it essential for self-service and enterprise data pipelines.

Core Capabilities (2026)

Data Connectivity

  • 100+ native connectors (databases, files, web, APIs, cloud services)
  • Recent data quick access from ribbon (preview)
  • Custom connectors & ODBC support (e.g., Vertica)

Transformation & Cleaning

  • No-code UI: split columns, pivot/unpivot, merge/append, group by
  • M language for advanced logic & custom functions
  • AI-assisted steps via Copilot (suggest transformations, explain code)

Refresh & Automation

  • Scheduled refreshes in Power BI Service / Fabric
  • Programmatic execution via REST API (preview)
  • Relative references in Dataflows Gen2

Integration & Governance

  • Seamless in Fabric, Power BI, Excel, Dataverse
  • Lineage & impact analysis via Microsoft Purview
  • Version control & sharing in Fabric workspaces

Best Practices for Enterprise Use (2026)

Design & Maintainability

  • Build reusable functions & parameters in M

Governance & Scalability

AI Leverage
