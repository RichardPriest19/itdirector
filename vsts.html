VSTS (Visual Studio Team Services)

Microsoft's legacy cloud-based DevOps platform (now evolved into Azure DevOps) for source control, CI/CD, agile planning, and team collaboration

Overview

VSTS was the original name for Microsoft's cloud DevOps offering (launched in 2015). It provided Git/TFVC repositories, build and release pipelines, work item tracking, and agile tools. In 2018, Microsoft rebranded VSTS to **Azure DevOps**.

Key Capabilities (VSTS Era)

  • Git and Team Foundation Version Control (TFVC)
  • Continuous Integration / Continuous Delivery (CI/CD) pipelines
  • Agile planning tools (backlogs, boards, sprints)
  • Test management and automated testing
  • Integrated wiki and team dashboards

Evolution to Azure DevOps

VSTS was renamed Azure DevOps in 2018. The platform is now available as:

  • Azure DevOps Services (cloud)
  • Azure DevOps Server (on-premises, formerly TFS)

Most organizations that still reference "VSTS" are actually using Azure DevOps today.

Relevance in 2026

While the name VSTS is largely retired, the concepts and many pipelines built during the VSTS era continue to run in Azure DevOps. Modern teams often migrate legacy VSTS projects to Azure DevOps or GitHub + GitHub Actions for better integration with the Microsoft ecosystem.
