Active Directory

Microsoft directory service for identity and access management

Overview

Active Directory (AD) is Microsoft's directory service that provides centralized user authentication and authorization for Windows networks. AD has been the standard identity management system in enterprises for decades, storing user accounts, computer accounts, groups, and security policies. With cloud adoption, Azure AD (now Microsoft Entra ID) extends identity management to the cloud, but on-premises Active Directory remains critical for managing Windows workstations and servers. Active Directory is the foundation of security in Microsoft-centric IT environments.

Critical Component: Active Directory is foundational to Microsoft infrastructure; compromising AD enables attackers to control the entire Windows environment.

Core Components

Domain Controller

  • Runs Active Directory service
  • Handles authentication and authorization
  • Stores directory database
  • Replicates to other domain controllers

Directory Database

  • User accounts and passwords
  • Computer accounts
  • Groups and distribution lists
  • Organizational units (OUs)

Group Policy

  • Centralized policy management
  • Computer and user policies
  • Security policy enforcement
  • Application deployment

Kerberos Authentication

  • Secure authentication protocol
  • Ticket-based system
  • Mutual authentication

Key Features

User Management

  • Create and manage user accounts
  • Password policies and expiration
  • Account lockout policies
  • Profile and home directory management

Computer Management

  • Register and manage computers
  • Group Policy application
  • Remote Group Policy updates
  • Computer certificate management

Group Management

  • Security groups for access control
  • Distribution groups for email
  • Group nesting
  • Dynamic group membership

Organizational Units (OUs)

  • Hierarchy for organizing objects
  • Group Policy scope
  • Delegation of administration

Active Directory Security

Authentication

  • Kerberos protocol
  • Password security
  • Credential caching
  • Multi-factor authentication integration

Authorization

  • Access control lists (ACLs)
  • Permissions on resources
  • Delegation of rights

Security Best Practices

  • Multiple domain controllers for resilience
  • Regular AD backups
  • Monitor for unauthorized changes
  • Privileged Access Management (PAM)
  • Periodic access reviews

Active Directory Threats

Common Attacks

  • Pass-the-hash attacks
  • Kerberoasting
  • Golden ticket attacks
  • Lateral movement through AD

Defense Strategies

  • Credential Guard on Windows 10+
  • Privileged Access Workstations
  • Constrained delegation
  • AD hardening tools

Hybrid Identity with Azure AD

Azure AD Connect

  • Synchronize on-premises AD to Azure AD
  • Hybrid identity model
  • Password hash sync or pass-through auth

Hybrid Environment Benefits
  • On-premises AD for Windows
  • Azure AD for cloud applications
  • Seamless user experience
